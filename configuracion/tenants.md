# Tenants

Tenants allow you to create isolated portals for different customers or internal teams, each with access only to the data and dashboards you explicitly grant them. Tenants are managed at **Admin → Tenants** in [[viewtimanager-overview|ViewtiManager]].

Use cases:
- **Managed service providers** — spin up portals for individual customers to add value and reduce churn.
- **Internal teams** — create separate tenants for different departments within your organization.

## Creating or Editing a Tenant

1. Navigate to **Admin → Tenants**.
2. Click **Add Tenant** (or select an existing tenant and click **Edit**).

### Tenant Details

| Field | Description |
|---|---|
| Name | Short, unique identifier for the tenant portal |
| Description | Optional free-form text describing the tenant's use case or owner |
| Created At | Read-only creation timestamp |
| Last Updated | Read-only last-modification timestamp |

## Sets Config (Data Access)

The Sets Config section controls which data sources each tenant can see and at what level of access:

| Column | Meaning |
|---|---|
| Move | Drag or click to reorder how data sets appear in the tenant portal |
| Set | Name of the data source (e.g., "DPI Records", "NetFlow Metrics") |
| Access | **Full** (view, query, export), **Partial** (filtered subset), or **None** (hidden) |
| Field | (Partial only) Schema field used to partition data (e.g., `customer_id`, `region`) |
| Type | (Partial only) `IP`, `IP Range`, `Subnet`, or `Identifier` |
| Values | (Partial only) Comma-separated list of allowed values matching the Type |

When using **Identifier** as the field type, you are referencing a decorated column generated by the [[vs-data-broker-overview|V.S. Data Broker]]. Checking the top-row Access checkbox applies the permission to all data sets at once.

## Tenant Users

The Tenant Users section lists which [[users|users]] belong to this tenant. To add or remove users, navigate to **Admin → Users**, select a user, and assign them to the desired tenant.

## Tenant Assignment for Users

When creating a user in [[users|the Users section]], the Tenant field specifies which tenant portal that user belongs to. Users without a specific tenant assignment use the default tenant (e.g., `Viewtinet`).

## Important Notes

- Always click **Save Changes** after modifying any tenant settings.
- Partial access with Identifier type requires the source field to be a decorated database column — consult the [[vs-data-broker-overview|V.S. Data Broker]] documentation for details.
- [[roles|Role sets]] further control which database tables each role can query, working alongside tenant data restrictions.
